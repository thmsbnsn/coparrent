# CoParrent Data Room Checklist

_Last updated: 2026-03-20_

This checklist is for building a real data room, not a folder full of random exports.

If an item here is missing, assume a serious buyer will notice.

## 1. Corporate And Ownership

- [ ] Seller legal name and operating identity
- [ ] Confirmation that BNSN Solutions owns the sale rights
- [ ] Confirmation that Thomas Benson is authorized to sell
- [ ] List of any contributors, contractors, or third parties with potential claims
- [ ] Signed assignment documents if outside contributors were involved
- [ ] Trademark, brand, logo, and domain ownership inventory
- [ ] Domain registrar details and transferability notes

If ownership is messy, price drops. Simple as that.

- [ ] One-page acquisition summary
- [ ] Product overview deck or memo
- [ ] Feature matrix
- [ ] Product screenshots
- [ ] Demo video
- [ ] Demo script

## 9. Financials

If there is no meaningful revenue yet, say that plainly. Do not dress a product asset up like a revenue-backed SaaS business.

- [ ] Revenue summary, if any

- [ ] Vendor spend
- [ ] Any paid acquisition or pilot costs
- [ ] Any projection assumptions used in buyer materials

## 10. Transition Package

- [ ] must-finish
- [ ] high-value expansions
- [ ] defer / skip

## 11. Red Flags To Resolve Before Opening Diligence

- [ ] Live frontend behind current repo
- [ ] Temporary QA exceptions still active
- [ ] Missing transfer documentation for any brand or account
- [ ] Unclear domain ownership
- [ ] Unclear contributor IP ownership
- [ ] Unsupported revenue claims
- [ ] Demo accounts not ready
- [ ] No recorded walkthrough

## Internal Notes

The more organized this is, the stronger the negotiating position.

For this asset, the data room matters more than usual. Without revenue carrying the story, the packaging has to carry more weight.
